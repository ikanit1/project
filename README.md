## Как работает CI/CD

### Файл `.github/workflows/ci.yml`

| Параметр | Значение | Описание |
|---|---|---|
| `on: push` | При каждом push | Тесты запускаются автоматически |
| `on: pull_request` | При каждом PR | Проверка перед слиянием веток |
| `runs-on: ubuntu-latest` | Ubuntu | Виртуальная машина GitHub |
| `python-version: "3.10"` | Python 3.10 | Версия интерпретатора |

### Шаги выполнения пайплайна

```
1. actions/checkout@v3     → Клонирование репозитория
2. actions/setup-python@v4 → Установка Python 3.10
3. pip install pytest       → Установка библиотеки тестирования
4. pytest                   → Запуск всех тестов
```

При ошибке в любом шаге — статус сборки становится **❌ failed**, коммит/PR помечается красным.

## Контрольные вопросы

**1. Что такое CI/CD?**  
CI/CD — практика автоматизации процессов разработки. CI (Continuous Integration) — автоматическая проверка кода при каждом изменении. CD (Continuous Delivery) — автоматическая подготовка к развёртыванию.

**2. В чём отличие CI от CD?**  
CI — интеграция и тестирование изменений. CD — доставка протестированного кода в production-среду.

**3. Для чего нужны автоматические тесты?**  
Для раннего обнаружения ошибок, проверки корректности кода и предотвращения регрессий при изменениях.

**4. Что делает GitHub Actions?**  
Запускает автоматические сценарии (workflows) при событиях в репозитории: push, pull_request и др.

**5. Что происходит при ошибке в тестах?**  
Сборка завершается со статусом `failed`, коммит помечается как неудачный. Разработчик получает уведомление об ошибке.
